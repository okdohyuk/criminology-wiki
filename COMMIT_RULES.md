# 범죄학 위키 커밋 규칙

이 문서는 범죄학 위키 저장소의 커밋 메시지 형식과 커밋 분할 기준을 정리한다.
기본적으로 상위 지침의 **Lore Commit Protocol**을 따르되, 이 위키에서는 **개발 작업 관점보다 위키 내용 관점**을 우선한다.

## 1. 핵심 원칙

- **커밋 메시지는 한국어로 작성한다.**
- **제목은 '무엇을 개발했다'가 아니라 '어떤 지식이 추가·통합·정정되었는가'를 먼저 드러낸다.**
- **한 커밋에는 하나의 학습 단위만 담는다.**
  - 한 이론 묶음
  - 한 인물 묶음
  - 한 범죄유형 묶음
  - 한 번의 링크 정정
  - 한 번의 목차/로그 갱신
- **가능하면 최대한 잘게 쪼갠다.**
  - 대형 카테고리 전체를 한 번에 묶지 말고, 개념적으로 가까운 소묶음으로 나눈다.
- **`raw/`는 읽기 전용 원본이므로 기본적으로 커밋하지 않는다.**
- **로컬 작업 흔적은 커밋하지 않는다.**
  - 예: `.obsidian/`, `.idea/`, `.claude/`, `.omx/`, `.DS_Store`

## 2. Prefix 규칙

커밋 제목은 반드시 아래 형식을 따른다.

```text
<prefix>(<scope>): <이번 커밋에 반영된 지식/정리 내용>
```

### 권장 prefix

- `feat`: 새 위키 페이지 추가, 새 내용 묶음 추가, 새 학습 축 형성
- `fix`: 내용 정정, 링크 복구, 메타/출처 오류 수정, 잘못된 분류 수정
- `chore`: 저장소 운영 정리, ignore, 위키 운영 규칙 파일, 구조 보조 작업
- `docs`: `wiki/log.md`, 설명 문서, 운영 문서, 안내 문구 정리
- `refactor`: 기존 내용을 삭제하지 않고 구조만 재배열했을 때 사용

### 권장 scope

- `repo`
- `rules`
- `위키구조`
- `범죄이론`
- `인물`
- `범죄유형`
- `형사정책`
- `피해자학`
- `범죄심리`
- `수사·사법`
- `통계·연구방법`
- `용어`
- `링크`

## 3. 제목줄 작성 규칙

제목은 **서술형 개발 문장**보다 **반영된 지식 묶음**을 보여주는 명사형/요약형 표현을 선호한다.

- 좋은 예: `feat(범죄이론): 고전주의 범죄학·합리적 선택·억제이론 기본 축`
- 좋은 예: `feat(피해자학): 피해자화·일상활동이론·생활양식-노출이론 연결`
- 좋은 예: `fix(링크): 스토킹·코헨·참여관찰 관련 깨진 위키링크 정리`
- 나쁜 예: `범죄이론을 추가한다`
- 나쁜 예: `파일 수정`
- 나쁜 예: `update wiki`

## 4. 메시지 본문 규칙

본문은 **왜 이 커밋이 필요한가**보다, **위키에 어떤 내용이 새로 들어오고 무엇과 결합되었는가**를 먼저 설명한다.

권장 순서:
1. 이번 커밋에 들어간 핵심 개념/페이지 묶음
2. 서로 어떤 축으로 연결되는지
3. 위키 탐색이나 학습 관점에서 왜 한 묶음인지

예:

```text
feat(범죄유형): 살인·강도·절도·사기 전통적 재산/폭력 범죄 축

전통적 범죄유형에 해당하는 핵심 페이지를 한 묶음으로 추가했다.
구성요건과 범죄유형론 허브를 함께 따라갈 수 있도록 기본 비교축을 맞췄다.

Category: 범죄유형
Source: 범죄학 강의 요약본
Confidence: high
Scope-risk: narrow
Reversibility: clean
Directive: 개별 범죄 페이지를 확장할 때는 범죄유형론 허브의 분류 문장도 함께 맞출 것
Tested: 파일 존재 및 메타 카테고리 확인
Not-tested: 세부 교차참조 전수 점검
```

## 5. Lore + 위키 전용 트레일러

상위 Lore 프로토콜을 유지하면서, 이 위키에서는 아래 트레일러를 권장한다.

- `Category:` 이번 커밋의 지식 분류 축
- `Source:` 반영한 원본 자료 묶음 또는 `없음`
- `Confidence:` 메시지와 분류의 신뢰 수준
- `Scope-risk:` 이 커밋이 위키 전체 구조에 미치는 범위
- `Reversibility:` 되돌리기 쉬운지 여부
- `Directive:` 이후 확장할 때 같이 봐야 할 허브/역링크/출처 주의사항
- `Tested:` 링크, 파일, 메타, 분류 등 확인한 항목
- `Not-tested:` 이번 커밋에서 확인하지 못한 부분

## 6. 권장 커밋 분할 기준

### A. 저장소/규칙
- `.gitignore`
- `COMMIT_RULES.md`
- 위키 운영 보조 문서

### B. 위키 구조
- `wiki/index.md`
- `wiki/log.md`
- 허브 페이지
- 카테고리 재분류

### C. 지식 페이지
가능하면 아래처럼 소묶음으로 자른다.
- 고전주의 축 / 사회해체 축 / 아노미 축
- 창시자 / 계승자 인물
- 전통적 범죄 / 특수범죄
- 피해자학 개론 / 피해위험 이론
- 연구설계 / 측정 / 윤리 사례
- 용어 기초

### D. 정정/점검
- 깨진 링크 수정
- 고아 페이지 복구
- 중복 페이지 통합
- 메타/출처 보강

## 7. 커밋 전 체크리스트

- [ ] prefix와 scope가 붙어 있는가?
- [ ] 제목이 개발 행위가 아니라 지식 내용 중심인가?
- [ ] 한 커밋이 한 학습 묶음으로 설명 가능한가?
- [ ] ignore 대상이 staging에 없는가?
- [ ] 새/수정 페이지에 출처와 메타가 있는가?
- [ ] `wiki/index.md`와 실제 파일명이 어긋나지 않는가?
- [ ] 교차참조가 필요한 경우 `[[위키링크]]`가 자연스럽게 연결되는가?
- [ ] `wiki/log.md`에 남길 변경이면 기록했는가?

## 8. 예시 제목 모음

- `feat(범죄이론): 사회해체이론·깨진 유리창 이론·집합효율성 지역사회 통제 축`
- `feat(인물): 베카리아·벤담 고전주의 형벌사상 계보`
- `feat(통계·연구방법): 인과관계·가설·변수·척도 측정 기초`
- `docs(위키구조): 전체 목차와 카테고리 탐색 경로`
- `fix(링크): 메스너·로젠펠드, 스토킹, 참여관찰 링크 정리`
- `chore(repo): 로컬 설정과 원본 폴더 ignore 규칙`
